Part 1: Ray Generation and Scene Intersection

How do we do ray generation and primitive intersection?

In our project, we are given normalized x and y positions in image space, and we convert them into a ray vector in camera space. We define the vector end using the following: x = tan(0.5 * hFov * PI / 180)*2*(x - 0.5), y = tan(0.5 * hFov * PI / 180)*2*(y - 0.5), and z = -1.

Then, we transform the vector end and origin (camera position) into world space using c2w (camera-to-world). This provides us with the ray we ray trace.

Our next step is to sample the positions into the buffer. For any given x and y coordinates, we add a sampling offset that is from 0 to 1, then normalize it. We generate the ray and take the average of n samples of the estimated global illuminance.

How does the triangle intersection algorithm work?

For the triangle intersection algorithm, we apply the Möller–Trumbore intersection algorithm. We set the ray equation equal to the weighted average of the triangle’s vertices based on barycentric coordinates. This provides us with the equation o + d*t = αv1 + βv2 + (1-α-β)v3. If there is a solution for α, β, and t, then there is a valid intersection. We solve the linear equations, then check that the answers are valid for our constraints of t > 0, α, β, and (1-α-β) are between 0 and 1.
